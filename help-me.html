Polvo API Routes Documentation

Method Route Request Info (Inputs) Response Info (Outputs) Notes
GET / No input Redirect
to /:id (if logged in)
or /welcome/auth		 Checks if user is logged in and redirects accordingly
GET /[slug] Params:
slug: string 		JSON Response: 
{
  user: {
    id: string,
    first_name: string,
    last_name: string,
    username: string,
    role: number,
    location: object | null,
    shop_name: string | null
  },
  posts: array,
  isSelf: boolean,
  profilePicture: string | null,
  pictureExt: string,
  userInstagramAuthed: boolean,
  oauthUrl: string | null
}
 Fetches profile page data for a specific user
POST /welcome/auth/login Form Data:
email: string
password: string 		Redirect
to /welcome/auth?status=success or /welcome/auth?status=failed		 Logs a user into native authentication
POST /welcome/auth/register Form Data:
username: string
email: string
password: string
passwordConfirm: string 		Redirect
to /welcome/auth?status=success or /welcome/auth?status=failed		 Registers a new user with native auth
GET /welcome/auth Query Params:
status: string 		JSON Response: 
{
  status: string
}
 Handles authentication status display page
GET /edit No input JSON Response: 
{
  user: {
    id: string,
    username: string,
    description: string,
    location: object | null
  },
  profilePicture: string | null,
  pictureExt: string
}
 Returns currently logged-in user's data for editing
POST /edit JSON Body:
username: string
description: string
location: string | object
role: string 		Redirect
to /:id?status=success or ?status=failed		 Updates the current user's profile information
GET /edit/autofill Query Params:
text: string 		JSON Response:
Autofill location suggestions from external service 		Calls external autofill service to complete location names
GET /users/ids Query Params:
id: string[] (multiple) 		JSON Response:
Array of user objects 
[
  {
    id: string,
    username: string,
    ...
  }
]
 Fetches multiple users by their IDs
GET /users/email Query Params:
email: string 		JSON Response:
Single user object 
{
  id: string,
  username: string,
  email: string,
  ...
}
 Fetches a user by their email address
GET /users/location Query Params:
loc: string[] (multiple, format: "lat,lng") 		JSON Response:
Array of user objects at those locations 		Fetches users at exact specified locations
POST /users/location JSON Body:
{ coords: { MaxLat: number, MinLat: number, MaxLong: number, MinLong: number } } 		JSON Response:
Array of user objects within the bounding box 		Fetches users within geographical coordinates (bounding box)
POST /avatar JSON Body:
extension: string
userid: string 		Plain Text
"Updated extension"		 Updates a user's avatar file extension
PUT /avatar No input Empty Response No-op placeholder endpoint
DEL /avatar No input Empty Response No-op placeholder endpoint
GET /userforpost Query Params:
userId: string 		JSON Response:
Single user object for posting context 		Fetches a user's data to pre-fill post forms
GET /auth/meta No input JSON Response: 
{
  state: string
}
 Generates a random state token for Meta OAuth flow
GET /auth/meta/callback Query Params:
code: string
state: string 		Redirect
to /:userId		 Completes Meta (Instagram) OAuth authentication
GET /auth/google No input Redirect
to Google Auth URL		 Initiates Google OAuth sign-in flow
GET /auth/google/callback Query Params:
code: string 		JSON Response:
{ id, first_name, last_name, email }		 Completes Google OAuth authentication
GET /[slug]/instagramposts Params:
slug: string
Query:
before: string | null
Header:
x-api-key: string 		JSON Response: 
{
  posts: [
    {
      id: string,
      caption: string,
      media_type: string,
      media_url: string,
      permalink: string,
      timestamp: string
    }
  ],
  cursor: string
}
 Fetches Instagram posts for a user
GET /[slug]/instagramposts/test No input Empty Response No-op placeholder endpoint
GET /avatar Query Params:
userId: string[] (multiple) 		JSON Response: 
{
  profilePictures: [string],
  extensions: {
    [userId]: string
  }
}
 Fetches avatar profiles and file extensions for multiple users
DEL /logout No input Plain Text
"Logged Out"		 Clears the authentication cookie and logs user out
GET /health No input Plain Text
"ok"		 Health check endpoint for monitoring
